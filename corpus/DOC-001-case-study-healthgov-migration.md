# Case Study: HealthGov Australia — Datacentre-to-Azure Migration

**Client:** HealthGov Australia (federal health services agency)
**Engagement:** Full datacentre exit and migration to Microsoft Azure
**Duration:** 14 months (Sep 2023 – Nov 2024)
**Contract value:** AUD 8.2M

## Scope

Migration of HealthGov's two on-premises datacentres to Azure, covering
**620 virtual machines** (Windows Server 2012 R2 and 2016), 41 SQL Server
instances, and 380 TB of data, with strict clinical-system availability
constraints.

## Approach

- Azure Migrate discovery and dependency mapping across all 620 VMs
- Wave-based migration: 11 waves grouped by application dependency
- Azure Site Recovery for replication; rehearsal cutovers for every wave
- Landing zone built to Australian Government ISM PROTECTED controls
- All workloads deployed to **Australia East** with DR in **Australia Southeast**

## Outcomes

- **Zero unplanned downtime** across all 11 migration waves
- Post-migration availability of **99.97%** measured over the first 6 months
  (against a contracted 99.95% SLA)
- Longest cutover window: 3 hours 20 minutes (contracted maximum: 4 hours)
- 23% reduction in infrastructure run cost in year one
- Decommissioning of both datacentres completed 6 weeks ahead of schedule
